# Заметки для ревью
Перечислил здесь известные мне ограничения моей реализации. Приоритетом была точность вёрстки по макету Figma.
## 1. Типографика захардкожена по Figma (нет Dynamic Type)
Размеры и цвета шрифтов заданы фиксированными значениями из макета, а не через системные текстовые стили. Это даёт точное соответствие Figma, однако текст не масштабируется под системный размер шрифта, поэтому на крупных устройствах / при увеличенном системном шрифте всё выглядит мелко.
К*ак решал бы:* перешел бы на `Font.custom(..., relativeTo:)` / системные text styles, чтобы шрифты соответствовали Dynamic Type.
## 2. Соотношение сторон карточки: 0.68 (Figma) -> 1.0 (фактически)
В макете соотношение сторон фото товара ≈**0.68**. Но многие изображения с бекенда при 0.68 выглядели не очень, часто не умещались в карточку. Поэтому в `DSProductCardImageView` соотношение изменено на **1.0.**
## 3. Предупреждения Accessibility Inspector -> Audit
Метки VoiceOver (`accessibilityLabel`/`value`/`traits`) расставлены по всему UI. Но аудит доступности всё ещё показывает три предупреждения.
1. **Отсутствие динамического текста (Dynamic Type).** Прямое следствие захардкоженной типографики (п. 1) — текст не реагирует на системный размер шрифта.
2. **Плохо читаемый текст значений веса товара.** Мелкий/низкоконтрастный текст веса продукта (взят из макета Figma).
3. **Мелкие области нажатия** у кнопок `+` / `−` (степпер) и кнопки «в избранное». По Figma иконки маленькие (сердечко 32 pt, значки степпера 16 pt), что меньше рекомендованных Apple 44×44 pt для тач-целей.
*Как решил бы:* Dynamic Type (п. 1) закрывает пункты 1–2, для пункта 3 — расширить тап-зону через `.frame(minWidth: 44, minHeight: 44)` / `.contentShape`, не меняя визуальный размер иконки. Всё это сознательно оставлено, чтобы вёрстка совпадала с макетом.
